# OpenCOOD数据准备与介绍详细教程

## 概述

OPV2V（Open Vehicle-to-Vehicle）数据集是OpenCOOD框架使用的主要数据集，专门设计用于协作感知研究。该数据集模拟了多个智能车辆（CAV）在不同场景下的协作感知任务，包含丰富的传感器数据和标注信息。

## 数据下载与准备

### 下载来源

所有数据都可以从Google Drive下载：
[https://drive.google.com/drive/folders/1dkDeHlwOVbmgXcDazZvO6TFEZ6V_7WUu](https://drive.google.com/drive/folders/1dkDeHlwOVbmgXcDazZvO6TFEZ6V_7WUu)

### 下载选项

1. **完整文件下载**：如果网络条件良好，可以直接下载完整的大型zip文件（如`train.zip`）

2. **分块下载**：如果下载大文件有困难，数据集也被分割成小块，可以在以`_chunks`结尾的目录中找到，如`train_chunks`

### 合并分块文件

下载分块文件后，使用以下命令将块合并：
```bash
cat train.zip.parta* > train.zip
unzip train.zip
```

### 目录结构

下载完成后，应将文件组织成以下结构：

```
OpenCOOD # OpenCOOD根目录
├── opv2v_data_dumping # 下载的opv2v数据
│   ├── train          # 训练数据
│   ├── validate       # 验证数据
│   ├── test           # 测试数据（CARLA默认城镇）
│   ├── test_culvercity # Culver City测试数据
├── opencood # 核心代码库
```

## 数据集结构详解

### 整体结构

OPV2V数据的组织结构如下：

```
opv2v_data_dumping
├── train # 训练数据
│   ├── 2021_08_22_21_41_24  # 场景文件夹
│     ├── data_protocol.yaml # 收集CARLA中数据所用的仿真参数
│     └──  1732 # 连接自动化车辆ID
│       └── 00000.pcd - 00700.pcd # 时间戳0到700的点云数据
│       ├── 00000.yaml - 00700.yaml # 每个时间戳对应的元数据
│       ├── 00000_camera0.png - 00700_camera0.png # 前置摄像头图像
│       ├── 00000_camera1.png - 00700_camera1.png # 右后摄像头图像
│       ├── 00000_camera2.png - 00700_camera2.png # 左后摄像头图像
│       └── 00000_camera3.png - 00700_camera3.png # 后置摄像头图像
├── validate  # 验证数据
├── test      # 测试数据
├── test_culvercity # Culver City测试数据
```

### 1. 数据划分

OPV2V数据集可分为4个不同文件夹：

- **`train`**：包含所有训练数据
  - 用于训练协作感知模型
  - 包含大量不同场景下的传感器数据和标注

- **`validate`**：用于训练期间的验证
  - 模型训练过程中用于监控性能
  - 帮助防止过拟合

- **`test`**：CARLA默认城镇的测试集
  - 标准测试集，用于评估模型性能
  - 与训练数据来自相同的虚拟环境

- **`test_culvercity`**：洛杉矶Culver City数字城镇的测试集
  - 在不同环境中测试模型泛化能力
  - 提供额外的挑战场景

### 2. 场景数据库

- **场景总数**：OPV2V共有73个场景
- **场景内容**：每个场景包含来自不同智能体在不同时间戳的数据流
- **命名规则**：每个场景按收集时间命名，例如：`2021_08_22_21_41_24`
  - 包含年月日时分秒信息
  - 便于追溯数据收集时间

### 3. 智能体内容

在每个场景文件夹下，当前场景中出现的每个智能体（即连接自动化车辆）的数据保存在不同文件夹中。

#### 文件夹结构
- 每个文件夹以智能体的唯一ID命名，如：1732
- 在每个智能体文件夹中，保存跨不同时间戳的数据
- 时间戳用5位整数作为文件名前缀表示（如：00700.pcd）

#### 智能体文件类型

智能体文件夹内包含三种类型的文件：

#### 3.1 激光雷达点云数据
- **文件格式**：`.pcd`文件
- **保存方式**：使用Open3d包保存
- **数据内容**：3D空间中的点云数据
- **时间序列**：从00000.pcd到序列结束的点云数据
- **用途**：主要传感器数据，用于3D目标检测和定位

#### 3.2 相机图像
- **相机配置**：每辆CAV配备4个RGB相机
  - `camera0`：前置摄像头
  - `camera1`：右后摄像头
  - `camera2`：左后摄像头
  - `camera3`：后置摄像头
- **视野覆盖**：4个摄像头捕捉周围场景360度视图
- **文件格式**：`.png`文件
- **参考**：可在[https://mobility-lab.seas.ucla.edu/opv2v/](https://mobility-lab.seas.ucla.edu/opv2v/)查看摄像头安装位置

#### 3.3 数据标注
- **文件格式**：`.yaml`文件
- **内容记录**：当前时间戳的重要信息

**自车信息（ego information）**：
- 当前自车姿态（带和不带GPS噪声）在CARLA世界坐标系下
- 自车速度（km/h）
- 激光雷达姿态
- 未来规划轨迹

**标定信息（calibration）**：
- 每个相机到激光雷达传感器的内参矩阵
- 每个相机到激光雷达传感器的外参矩阵
- 用于多模态数据融合

**对象标注（objects annotation）**：
- 每个周围人类驾驶车辆的姿态和速度
- 条件：至少有一个点被智能体的激光雷达传感器击中
- 详见[data annotation section](data_annotation_tutorial.md)

### 4. 数据收集协议

除了智能体内容，每个场景数据库还有一个名为`data_protocol.yaml`的YAML文件。

#### 协议文件内容
- **仿真配置**：记录收集当前场景的仿真配置
- **用途**：
  - 用于日志回放数据
  - 使用户能够在不改变原始事件的情况下添加/修改传感器以适应新任务

#### 数据收集代码
为了帮助用户在CARLA中收集自定义数据（如不同的传感器配置和模态），具有相似数据格式和结构的OPV2V数据收集代码已发布在[OpenCDA](https://github.com/ucla-mobility/OpenCDA)的`feature/data_collection`分支中。

用户可以参考其[文档](https://opencda-documentation.readthedocs.io/en/latest/md_files/introduction.html)获取详细说明。

## 数据集特点与优势

### 1. 多模态数据
- 同时提供激光雷达点云和相机图像
- 支持多模态融合研究
- 丰富的传感器数据类型

### 2. 协作感知场景
- 多个智能体同时采集数据
- 模拟真实的V2V协作场景
- 支持不同融合策略研究

### 3. 精确标注
- 详细的3D边界框标注
- 完整的姿态和速度信息
- 传感器间精确标定

### 4. 大规模数据
- 73个不同场景
- 长时间序列数据
- 丰富的交通场景

## 数据预处理建议

在使用OPV2V数据集时，通常需要进行以下预处理：

1. **点云预处理**：滤波、去噪、坐标变换
2. **数据同步**：确保不同传感器数据的时间同步
3. **坐标系统一**：将所有传感器数据转换到统一坐标系
4. **数据增强**：根据需要进行数据增强操作

## 使用注意事项

1. **存储空间**：数据集体积庞大，需确保足够的存储空间
2. **数据完整性**：下载后检查数据完整性
3. **许可证**：遵守数据使用许可证要求
4. **预处理**：根据具体任务需求进行适当的数据预处理

OPV2V数据集为协作感知研究提供了高质量、大规模的实验平台，支持多种算法和方法的开发与评估。
