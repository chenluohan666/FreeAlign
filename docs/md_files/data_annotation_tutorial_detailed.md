# OpenCOOD数据标注系统详细教程

## 概述

OpenCOOD框架将所有真值标注按智能体和时间戳分别保存在YAML文件中。这种设计提供了丰富的传感器数据和环境标注，支持协作感知算法的开发和评估。

## 标注文件结构

### 文件路径命名

标注文件路径格式为：`{scenario_id}/{agent_id}/{timestamp}.yaml`

例如：`2021_08_24_21_29_28/4805/000069.yaml`
- `2021_08_24_21_29_28`：场景ID（收集数据的时间）
- `4805`：智能体ID
- `000069`：时间戳69

这表示在场景`2021_08_24_21_29_28`中，智能体4805在时间戳69的数据标注，以该智能体的视角为准。

## 详细标注内容解析

### 1. 相机参数（Camera Parameters）

#### camera0（前置摄像头）
```yaml
camera0: # 前置摄像头参数
  cords: # CARLA地图坐标系下的x,y,z,roll,yaw,pitch
  - 141.35067749023438  # x坐标
  - -388.642578125       # y坐标
  - 1.0410505533218384   # z坐标
  - 0.07589337974786758  # roll角
  - 174.18048095703125   # yaw角
  - 0.20690691471099854  # pitch角
  
  extrinsic: # 从摄像头到激光雷达的外参矩阵
  - - 0.9999999999999999      # 旋转矩阵元素[0,0]
    - -5.1230071481984265e-18 # 旋转矩阵元素[0,1]
    - 9.322129061605055e-20   # 旋转矩阵元素[0,2]
    - -2.999993025731527      # 平移向量元素[0,3]
  - - -2.5011383190939924e-18 # 旋转矩阵元素[1,0]
    - 1.0                     # 旋转矩阵元素[1,1]
    - 1.1458579204685086e-19  # 旋转矩阵元素[1,2]
    - -3.934422863949294e-06  # 平移向量元素[1,3]
  - - 2.7713237218713775e-20  # 旋转矩阵元素[2,0]
    - 3.7310309839064755e-20  # 旋转矩阵元素[2,1]
    - 1.0                     # 旋转矩阵元素[2,2]
    - 0.8999999040861146      # 平移向量元素[2,3]
  - - 0.0                     # 齐次坐标[3,0]
    - 0.0                     # 齐次坐标[3,1]
    - 0.0                     # 齐次坐标[3,2]
    - 1.0                     # 齐次坐标[3,3]
  
  intrinsic: # 相机内参矩阵
  - - 335.639852470912  # fx（x方向焦距）
    - 0.0               # 偏斜系数
    - 400.0             # cx（x方向主点）
  - - 0.0               # 偏斜系数
    - 335.639852470912  # fy（y方向焦距）
    - 300.0             # cy（y方向主点）
  - - 0.0               # 畸变参数
    - 0.0               # 畸变参数
    - 1.0               # 齐次坐标
```

- **cords**：描述相机在CARLA地图坐标系下的6自由度姿态（位置和姿态角）
- **extrinsic**：4x4外参矩阵，定义从相机坐标系到激光雷达坐标系的变换
- **intrinsic**：3x3内参矩阵，包含焦距和主点信息

#### 其他摄像头
- `camera1`：右后摄像头参数
- `camera2`：左后摄像头参数
- `camera3`：后置摄像头参数

每个摄像头都包含相同的参数结构（位置、外参矩阵、内参矩阵）。

### 2. 智能体状态信息（Agent State）

#### 速度信息
```yaml
ego_speed: 18.13 # 智能体当前速度，单位km/h
```
- 记录智能体在当前时间戳的行驶速度

#### 激光雷达姿态
```yaml
lidar_pose: # 激光雷达在CARLA地图坐标系下的姿态
- 144.33  # x坐标
- -388.94 # y坐标
- 1.93    # z坐标
- 0.078   # roll角
- 174.18  # yaw角
- 0.21    # pitch角
```
- 激光雷达传感器在全局坐标系中的精确位置和姿态

#### 规划轨迹
```yaml
plan_trajectory: # 智能体的规划轨迹
- - 140.  # 轨迹点1的x坐标
  - -388  # 轨迹点1的y坐标
  - 87    # 轨迹点1的时间戳
```
- 智能体的预期行驶轨迹，用于路径规划和预测

#### 定位信息
```yaml
predicted_ego_pos: # 通过GPS获得的智能体定位(x,y,z,roll,yaw,pitch)
- 143.78  # x坐标
- -388.94 # y坐标
- 0.036   # z坐标
- 0.080   # roll角
- -185.95 # yaw角
- 0.18    # pitch角

true_ego_pos: # 智能体的真实定位
- 143.83  # x坐标
- -388.89 # y坐标
- 0.032   # z坐标
- 0.075   # roll角
- 174.18  # yaw角
- 0.21    # pitch角
```
- **predicted_ego_pos**：含GPS噪声的定位信息，模拟真实GPS的不准确性
- **true_ego_pos**：无噪声的真实定位信息，用于评估和基准测试

### 3. 周围车辆标注（Surrounding Vehicles）

```yaml
vehicles: # 至少有一个激光雷达点被该智能体击中的周围车辆
  4796: # 车辆ID（即对象ID）
    angle: # CARLA地图坐标系下的roll, yaw, pitch
    - 0.096   # roll角
    - -177.86 # yaw角
    - 0.197   # pitch角
    
    center: # 从边界框中心到车辆前轴的相对位置
    - 0.0004 # x方向偏移
    - 0.0005 # y方向偏移
    - 0.71   # z方向偏移
    
    extent: # 车辆的半长、半宽、半高（米）
    - 2.45 # 半长度（米）
    - 1.06 # 半宽度（米）
    - 0.75 # 半高度（米）
    
    location: # 车辆前轴中心在CARLA地图坐标系下的位置
    - 158.55 # x坐标
    - -385.75 # y坐标
    - 0.032   # z坐标
    
    speed: 19.47 # 车辆速度
    
  4880: # 另一个车辆的标注
    # ... 其他车辆的相同结构标注
```

#### 车辆标注详细说明：

- **车辆ID**：每个被检测车辆的唯一标识符
- **angle**：车辆在全局坐标系下的姿态角（roll, yaw, pitch）
- **center**：边界框中心相对于车辆前轴的偏移量
- **extent**：车辆尺寸的半值（长/2, 宽/2, 高/2），单位米
- **location**：车辆前轴中心的全局坐标
- **speed**：车辆的行驶速度

## 数据标注特点与优势

### 1. 多模态标注
- 同时提供多个摄像头的内外参
- 激光雷达坐标系与相机坐标系之间的精确变换
- 支持多传感器融合算法开发

### 2. 高精度标注
- 提供车辆的精确3D边界框（位置、姿态、尺寸）
- 详细的传感器标定信息
- 高频率的时间同步数据

### 3. 真实性模拟
- 包含GPS噪声的定位信息，模拟真实环境
- 真实的车辆动力学信息
- CARLA仿真环境下的精确物理模拟

### 4. 协作感知支持
- 记录多个智能体视角下的相同场景
- 支持V2V通信和协作算法研究
- 提供车辆间相对位置信息

## 使用标注数据的注意事项

### 1. 坐标系转换
- 不同传感器使用不同的坐标系
- 需要使用外参矩阵进行坐标系转换
- 确保所有数据在统一坐标系下

### 2. 时间同步
- 确保使用相同时间戳的数据
- 考虑传感器采集延迟
- 处理数据丢失或异常值

### 3. 数据验证
- 检查车辆标注的完整性
- 验证传感器数据的合理性
- 处理边界情况（如物体部分可见）

### 4. 性能评估
- 使用真值数据进行模型性能评估
- 区分含噪声数据和真值数据的使用场景
- 考虑不同传感器的特性和局限性

## 应用场景

这些详细的标注数据支持多种研究应用：
- 3D目标检测算法开发
- 多传感器融合算法研究
- V2V协作感知算法
- 定位与地图构建
- 路径规划与预测
- 传感器标定与校准

数据标注系统为协作感知研究提供了高质量、精确的基准数据，支持从算法开发到性能评估的完整研究流程。
